## How Configuration Works

Q-ez uses Dart's built-in `--dart-define` mechanism for environment configuration. There is no `.env` file. Values are compiled into the app at build time and accessed via `String.fromEnvironment` / `bool.fromEnvironment`.

```dart
// lib/core/config/firebase_config.dart
class Environment {
  static const String firebaseApiKey =
      String.fromEnvironment('FIREBASE_API_KEY');

  static const String firebaseProjectId =
      String.fromEnvironment('FIREBASE_PROJECT_ID');

  static const bool isProduction =
      bool.fromEnvironment('PRODUCTION', defaultValue: false);
}
```

For platform-specific Firebase configuration (Android, iOS, Web), values are stored in `lib/core/config/firebase_options.dart`, which is generated by the FlutterFire CLI and should not be committed to public repositories.

## Environment Variables

### Application Variables

| Variable | Type | Default | Required | Description |
|----------|------|---------|----------|-------------|
| `FIREBASE_API_KEY` | `String` | — | Yes (web) | Firebase Web API key. Used for web platform initialization. Android and iOS use their native config files instead. |
| `FIREBASE_PROJECT_ID` | `String` | — | Yes | Firebase project ID. Used to identify the Firebase project across all platforms. |
| `PRODUCTION` | `bool` | `false` | No | When `true`, enables production mode: Crashlytics error reporting is active, verbose debug logging is suppressed. |

### Example Usage

**Development (local run):**

```bash
flutter run \
  --dart-define=FIREBASE_API_KEY=AIzaSyAbc123ExampleKey \
  --dart-define=FIREBASE_PROJECT_ID=q-ez-dev \
  --dart-define=PRODUCTION=false
```

**Production build:**

```bash
flutter build appbundle --release \
  --dart-define=FIREBASE_API_KEY=AIzaSyXyz789ProductionKey \
  --dart-define=FIREBASE_PROJECT_ID=q-ez-production \
  --dart-define=PRODUCTION=true
```

## Firebase Configuration Fields

Firebase configuration is stored in `lib/core/config/firebase_options.dart`. This file is generated by the FlutterFire CLI (`flutterfire configure`) and contains platform-specific values.

### Android Configuration

| Field | Description | Example |
|-------|-------------|---------|
| `apiKey` | Android API key | `AIzaSyAbc123...` |
| `appId` | Android app ID | `1:123456789:android:abc123` |
| `messagingSenderId` | FCM sender ID | `123456789012` |
| `projectId` | Firebase project ID | `q-ez-production` |
| `storageBucket` | Cloud Storage bucket | `q-ez-production.appspot.com` |

### iOS Configuration

| Field | Description | Example |
|-------|-------------|---------|
| `apiKey` | iOS API key | `AIzaSyDef456...` |
| `appId` | iOS app ID | `1:123456789:ios:def456` |
| `messagingSenderId` | FCM sender ID | `123456789012` |
| `projectId` | Firebase project ID | `q-ez-production` |
| `storageBucket` | Cloud Storage bucket | `q-ez-production.appspot.com` |
| `iosBundleId` | iOS bundle identifier | `com.qez.q_ez` |

### Web Configuration

| Field | Description | Example |
|-------|-------------|---------|
| `apiKey` | Web API key | `AIzaSyGhi789...` |
| `appId` | Web app ID | `1:123456789:web:ghi789` |
| `messagingSenderId` | FCM sender ID | `123456789012` |
| `projectId` | Firebase project ID | `q-ez-production` |
| `storageBucket` | Cloud Storage bucket | `q-ez-production.appspot.com` |
| `authDomain` | Auth domain | `q-ez-production.firebaseapp.com` |
| `measurementId` | Google Analytics ID | `G-XXXXXXXXXX` |

### Platform Config Files

| Platform | File | Location |
|----------|------|----------|
| Android | `google-services.json` | `android/app/google-services.json` |
| iOS | `GoogleService-Info.plist` | `ios/Runner/GoogleService-Info.plist` |
| Web | Inline in `firebase_options.dart` | `lib/core/config/firebase_options.dart` |

An example Android config is provided at `android/app/google-services.json.example`. Copy it and replace placeholder values with your actual Firebase project values.

## Build Environments

Q-ez has two logical environments: **development** and **production**.

### Development

Used for local development and testing.

| Setting | Value |
|---------|-------|
| `PRODUCTION` | `false` |
| Firebase project | Separate dev project (e.g., `q-ez-dev`) |
| Crashlytics | Disabled |
| Debug logging | Enabled (verbose) |
| Firestore persistence | Enabled |

### Production

Used for App Store / Play Store / Firebase Hosting releases.

| Setting | Value |
|---------|-------|
| `PRODUCTION` | `true` |
| Firebase project | Production project (e.g., `q-ez-production`) |
| Crashlytics | Enabled |
| Debug logging | Disabled |
| Firestore persistence | Enabled |
| Code minification (Android) | Enabled |
| Resource shrinking (Android) | Enabled |

### Switching Environments

There are no build flavors configured in the project. Environment switching is done entirely through `--dart-define` flags. To target a different Firebase project, run `flutterfire configure` again and select the desired project, or manually update `firebase_options.dart`.

## Supplying Variables at Runtime

### Local Development

Pass `--dart-define` flags directly to `flutter run`:

```bash
flutter run \
  --dart-define=FIREBASE_API_KEY=your_key \
  --dart-define=FIREBASE_PROJECT_ID=your_project \
  --dart-define=PRODUCTION=false
```

### VS Code

Add a launch configuration in `.vscode/launch.json`:

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Q-ez (dev)",
      "request": "launch",
      "type": "dart",
      "args": [
        "--dart-define=FIREBASE_API_KEY=your_key",
        "--dart-define=FIREBASE_PROJECT_ID=q-ez-dev",
        "--dart-define=PRODUCTION=false"
      ]
    },
    {
      "name": "Q-ez (production)",
      "request": "launch",
      "type": "dart",
      "args": [
        "--dart-define=FIREBASE_API_KEY=your_prod_key",
        "--dart-define=FIREBASE_PROJECT_ID=q-ez-production",
        "--dart-define=PRODUCTION=true"
      ]
    }
  ]
}
```

### Android Studio / IntelliJ

1. Open **Run > Edit Configurations**.
2. Select your Flutter run configuration.
3. Add to **Additional run args**:
   ```
   --dart-define=FIREBASE_API_KEY=your_key --dart-define=FIREBASE_PROJECT_ID=your_project
   ```

### Using a dart-define File (Flutter 3.17+)

For convenience, you can store dart-define values in a JSON file and reference it:

Create `config/dev.json`:

```json
{
  "FIREBASE_API_KEY": "your_dev_api_key",
  "FIREBASE_PROJECT_ID": "q-ez-dev",
  "PRODUCTION": "false"
}
```

Then run:

```bash
flutter run --dart-define-from-file=config/dev.json
```

> Add `config/dev.json` and `config/production.json` to `.gitignore` to avoid committing secrets.

## CI/CD Configuration

In CI/CD pipelines, pass environment variables as pipeline secrets and inject them into the build command.

### GitHub Actions

```yaml
- name: Build Web
  run: |
    flutter build web --release \
      --dart-define=FIREBASE_API_KEY=${{ secrets.FIREBASE_API_KEY }} \
      --dart-define=FIREBASE_PROJECT_ID=${{ secrets.FIREBASE_PROJECT_ID }} \
      --dart-define=PRODUCTION=true
```

### Android Signing Variables

The Android build also requires keystore credentials. These are read from environment variables when `android/keystore.properties` is absent:

| Variable | Description |
|----------|-------------|
| `KEY_ALIAS` | Keystore key alias |
| `KEY_PASSWORD` | Key password |
| `STORE_FILE` | Absolute path to the keystore file |
| `STORE_PASSWORD` | Keystore store password |

See `android/keystore.properties.example` for the local development equivalent.

## Sensitive Files and .gitignore

The following files contain secrets and must not be committed to version control:

| File | Contents | Template Available |
|------|----------|-------------------|
| `android/app/google-services.json` | Android Firebase config | `google-services.json.example` |
| `ios/Runner/GoogleService-Info.plist` | iOS Firebase config | — |
| `lib/core/config/firebase_options.dart` | All platform Firebase config | — |
| `android/keystore.properties` | Android signing credentials | `keystore.properties.example` |
| `android/app/release.keystore` | Android signing keystore | — |
| `config/dev.json` | dart-define values (if used) | — |
| `config/production.json` | dart-define values (if used) | — |

These entries are included in `.gitignore`. Verify they are not tracked before pushing to a remote repository:

```bash
git status --short
```

None of the files listed above should appear as untracked or modified files in a clean repository.
